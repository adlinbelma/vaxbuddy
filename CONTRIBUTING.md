# Contributing to VaxBuddy

Thank you for your interest in contributing to VaxBuddy! Contributions are welcome from everyone.

**Repository:** https://github.com/adlinbelma/vaxbuddy

## Code of Conduct

Be respectful, inclusive, and constructive in all interactions.

## How to Contribute

### Reporting Bugs
1. Check if the issue already exists
2. Provide a clear description
3. Include steps to reproduce
4. Add screenshots if applicable
5. Mention your browser/device

### Suggesting Features
1. Check if already suggested
2. Explain the use case
3. Describe the expected behavior
4. Include mockups if helpful

### Submitting Changes
1. Fork the repository
2. Create a descriptive branch name
3. Make your changes
4. Test thoroughly
5. Write clear commit messages
6. Open a Pull Request

## Development Guidelines

### Code Style
- Keep it simple and readable
- Use meaningful variable names
- Comment complex logic
- Follow existing patterns

### Testing
- Test all major features before submitting
- Test on mobile and desktop
- Check different browsers
- Verify database changes work correctly

### Commit Messages
- Use present tense ("Add feature" not "Added feature")
- Be specific about what changed
- Keep first line under 50 characters

### Pull Request Process
1. Update README if needed
2. Add description of changes
3. Link related issues
4. Wait for review
5. Address feedback
6. Merge when approved
